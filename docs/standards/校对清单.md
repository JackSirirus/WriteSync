# 校对清单

> WriteSync（共笔）项目
> 适用阶段：每章终版定稿前的最后校对（v0.3.0 后简化为 2 调用/章）
> 当前工作流：文笔 Agent → 校对 Agent

---

## 一、错别字检查

### 1.1 常见错别字

- [ ] 的/地/得混淆（"飞快地跑" vs "跑的很快"）
- [ ] 在/再混淆（"在那里" vs "再次"）
- [ ] 做/作混淆（"做事" vs "作业"）
- [ ] 象/像混淆（"好像" vs "大象"）
- [ ] 竟/竟混淆（"竟然" vs "究竟"）
- [ ] 即/既混淆（"即使" vs "既然"）
- [ ] 副/幅混淆（"一副手套" vs "一幅画"）
- [ ] 了/了（多字/少字）
- [ ] 同音字错误（"账户" vs "帐号"，"登录" vs "登陆"）
- [ ] 网文常用错别字（"筑基" vs "铸基"，"金丹" vs "金单"）

### 1.2 人名/地名/术语一致性

- [ ] 所有角色名字前后一致（不出现"李明"变"李铭"）
- [ ] 地名前后一致
- [ ] 境界/等级名称前后一致
- [ ] 物品/道具名称前后一致

---

## 二、标点符号规范

### 2.1 句号、逗号、顿号

- [ ] 句子结束使用句号（。）而非英文句号（.）
- [ ] 列表项之间使用顿号（、）或分号（；），不使用逗号混淆
- [ ] 逗号用于分隔句子内部的部分，不滥用

### 2.2 引号

- [ ] 中文对话使用双引号（「」或""），不混用
- [ ] 引号嵌套时区分内引号和外引号
- [ ] 引号内的停顿符合正常说话节奏

### 2.3 省略号与破折号

- [ ] 使用中文省略号（……）而非英文（...）
- [ ] 使用中文破折号（——）而非英文短横线（--）
- [ ] 破折号用于解释、转折、声音延长，不滥用

### 2.4 感叹号与问号

- [ ] 感叹号不过度使用（连续多个感叹号降低表达力度）
- [ ] 反问句使用问号而非感叹号
- [ ] 语气词后的标点符合语气强度

---

## 三、格式规范

### 3.1 段落格式

- [ ] 每个段落有且只有一个核心意思（不一段落包含过多内容）
- [ ] 自然段之间有适当分隔
- [ ] 章节开头无需空两格
- [ ] 对话段落缩进格式统一

### 3.2 章节格式

- [ ] 章节标题格式统一（如"第一章 觉醒"）
- [ ] 章节之间有过渡（留空或分隔符）
- [ ] 每章字数在平台要求范围内（通常 3000-5000 字）

### 3.3 数字使用

- [ ] 阿拉伯数字 vs 中文数字使用统一（"3000字" vs "三千字"）
- [ ] 年份使用正确格式（"2024年"，不写"二零二四年"）
- [ ] 百分比使用正确格式（"80%"，"百分之八十"）

---

## 四、敏感内容格式规范

> 注：内容层面的合规判断由检查 Agent 负责（见《检查 Agent 评估标准》5.3），本层只负责格式规范。

### 4.1 政治敏感格式

- [ ] 真实地名/机构名是否已替换为虚构名称
- [ ] 涉及政治的描述是否使用了虚构背景

### 4.2 亲密场景格式

- [ ] 亲密场景是否使用省略/暗示格式，不写具体描写
- [ ] 身体部位描写是否使用模糊化表达

### 4.3 暴力场景格式

- [ ] 暴力场景是否使用"..."省略，不写详细过程
- [ ] 血腥描写是否点到为止，不渲染恐怖细节

---

## 五、对话格式

### 5.1 对话标签

- [ ] 说话人在前/中/后的格式统一
- [ ] 动作描写与对话标签的格式统一
- [ ] 不滥用"说道"、"应道"等替换词

### 5.2 对话语气

- [ ] 对话语气符合角色性格和身份
- [ ] 不同角色的对话有区分度（不千人一口）
- [ ] 网文特有的"系统提示音"/"面板显示"格式统一

### 5.3 内心独白

- [ ] 内心独白格式统一（"……"或"（心想：……）"）
- [ ] 内心独白与对话比例合理（不过度堆砌）

---

## 六、专有名词格式

### 6.1 网文特有格式

- [ ] 系统面板/属性面板格式统一
- [ ] 境界/等级名称格式统一
- [ ] 技能/功法名称格式统一（加书名号/引号/不加，统一规则）
- [ ] 货币/积分等名称格式统一

### 6.2 力量体系格式

- [ ] 境界等级用词统一（"筑基境"还是"筑基期"，全书一致）
- [ ] 境界突破的描述格式统一
- [ ] 力量数值表述统一

---

## 七、检查输出格式

校对 Agent 检查完成后，输出以下结构：

```markdown
## 校对报告

### 该章基本信息
- 章节：第N章
- 字数：XXXX字
- 当前阶段：节奏稿

### 发现的问题

#### 🔴 严重问题（必须修改）
- [问题描述]
  - 位置：第X段/第X行
  - 当前内容：[原文]
  - 修改为：[建议修改]

#### 🟡 中等问题（建议修改）
- [问题描述]
  - 位置：第X段/第X行
  - 当前内容：[原文]
  - 修改为：[建议修改]

#### 🟢 轻微问题（可选修改）
- [问题描述]
  - 建议修改：

### 通过的检查项
- ✅ [通过的检查项]

### 敏感内容格式检查
- [ ] 通过
- ⚠️ 发现格式问题：[列表]

### 总体评价
[是否可以发布，给出结论]

### 修改后字数预估
- 修改后预计字数：XXXX字
```

---

## 八、特殊说明

1. **校对是最后一道工序**，不修改内容逻辑和叙事节奏，只改错别字、标点、格式
2. **校对发现问题应标记，不直接修改**，除非是明显的笔误
3. **格式层面的敏感内容合规由校对 Agent 负责，内容层面的合规判断由检查 Agent 负责**
4. **标点符号以《标点符号用法》（GB/T 15834）为准**
