# 检查 Agent 评估标准

> WriteSync（共笔）项目
> 来源：Craft Companion 终版自查清单
> 适用阶段：每章初稿完成后，作为**人工参考**清单辅助审阅（v0.3.0 后编辑 Agent 已合并，不再独立调用）
> 使用说明：作者/审阅者参考本清单逐项核验草稿质量

---

## 一、设定层

检查世界观、数值、机制、时间线的一致性。

### 1.1 力量体系一致性

- [ ] 力量等级是否清晰、不矛盾？
- [ ] 能力设定是否有代价/限制？是否前后一致？
- [ ] 力量差距是否合理（高境界 vs 低境界）？
- [ ] 主角修炼/升级的数值是否自洽（境界名称/等级数量）？

### 1.2 地理与空间一致性

- [ ] 地点之间的距离感是否合理？（不能前一章在城市，下一章就到了千里之外）
- [ ] 地理描述前后是否一致？
- [ ] 空间转换是否有交代（传送/赶路/时间流逝）？

### 1.3 时间线一致性

- [ ] 事件发生顺序是否通顺？
- [ ] 昼夜交替是否合理（连续战斗后是否需要休息）？
- [ ] 关键时间节点是否对得上（节日/季节/特殊日期）？

### 1.4 规则一致性

- [ ] 物品/道具的能力是否前后一致？
- [ ] 设定好的规则是否被随意打破（作者临时开金手指）？
- [ ] 系统面板/属性面板的数值变化是否有记录？

---

## 二、人物层

检查人物行为是否符合 OOC（Out Of Character）、性格渐进性、知识边界。

### 2.1 性格一致性（OOC 检查）

- [ ] 主角行为是否与预设性格一致？
- [ ] 反派是否有独立的性格逻辑，不是单纯的"工具人"？
- [ ] 配角性格是否稳定（不因剧情需要突然变聪明/变蠢）？
- [ ] 情绪反应是否合理（面对同一件事，不同性格的人反应不同）？

### 2.2 渐进性成长

- [ ] 主角的能力/性格变化是否有迹可循（不是突变）？
- [ ] 成长是否与经历挂钩（经历事件 → 能力提升/性格改变）？
- [ ] 成长节奏是否合理（升级太快失去张力，太慢失去爽感）？

### 2.3 知识边界

- [ ] 主角知道的内容不超过其经历范围（不能突然知道不该知道的事）？
- [ ] 配角的认知是否受其身份限制（普通人不知道修炼知识）？
- [ ] 对话中的专业术语是否与角色身份匹配？

### 2.4 人物弧线追踪

- [ ] 该章是否推进了预设的角色弧线？
- [ ] 角色变化是否有伏笔/铺垫？
- [ ] 角色决定是否符合其当前弧线阶段？

---

## 三、结构层

检查承接性、篇幅分配、信息渗透方式。

### 3.1 承接性

- [ ] 该章开头是否承接了上章结尾（不能上一章在战斗，下一章在喝茶）？
- [ ] 章节内的场景转换是否有过渡？
- [ ] 倒叙/插叙是否有明确标识（时间跳转交代清楚）？

### 3.2 篇幅分配

- [ ] 该章的核心事件是否突出（不是被支线冲淡）？
- [ ] 打斗/修炼/日常的比例是否合适（网文读者期待节奏）？
- [ ] 章节长度是否符合平台要求（起点通常 3000-5000 字/章）？

### 3.3 信息渗透（No Info-Dump）

- [ ] 世界观信息是否通过剧情自然带出，而不是旁白讲解？
- [ ] 设定介绍是否避免"课堂模式"（A对B说："我来给你讲讲这个世界的规则……"）？
- [ ] 力量体系/背景信息是否在冲突场景中自然展现？

### 3.4 章节结构

- [ ] 该章是否有完整的叙事骨架（起承转合至少各有一个节点）？
- [ ] 核心事件是否在本章内有明确推进？

> 注：高潮质量、钩子设计、断章位置由节奏 Agent 负责优化，本层只确保骨架完整。

---

## 四、文字层

检查叙事效率、单句成段、情绪动作、重复冗余。

### 4.1 叙事效率

- [ ] 无效描写是否过多（读者不关心的环境描写占篇幅）？
- [ ] 内心独白是否精简（不是大段心理活动）？
- [ ] 重复表达是否过多（同一个意思换多种说法）？

### 4.2 单句成段

- [ ] 是否避免"一句话憋一个长段落"？
- [ ] 短段落是否用于营造节奏感（快速推进时）？
- [ ] 长段落是否用于情绪铺垫（关键场景时）？

### 4.3 情绪动作去重

- [ ] 同一个情绪是否用不同动作/反应表达（不是反复说"他很愤怒"）？
- [ ] 战斗场景的描写是否有变化（避免一招反复描写）？
- [ ] 表情描写是否避免重复（不反复写"嘴角勾起"、"眼中闪过"）

### 4.4 语法与标点

- [ ] 标点使用是否规范？
- [ ] 是否有错别字？
- [ ] 对话格式是否统一（谁说的话、说话方式是否一致）？
- [ ] 引号使用是否正确？

---

## 五、网文专项检查

针对网文平台的特殊要求。

### 5.1 爽点检查

- [ ] 该章是否有明确的爽点（装逼/打脸/升级/收获/情感满足）？

> 注：爽点铺垫质量、爽点释放节奏、高潮设计由节奏 Agent 负责优化，本层只确保该章存在爽点。

### 5.2 毒点检查

- [ ] 主角是否有憋屈情节（被欺负不反抗/反复原谅）？
- [ ] 是否有圣母行为（无底线原谅敌人）？
- [ ] 是否有绿帽/被背叛风险情节？
- [ ] 是否有明显的憋屈憋气段落（让读者想弃书）？

### 5.3 审核合规

- [ ] 是否有政治/军事敏感内容？
- [ ] 是否有露骨的色情描写？
- [ ] 是否有过度血腥暴力细节？
- [ ] 是否使用了现实地名/真实人物？
- [ ] 敏感词是否已替换？

### 5.4 叙事完整性（为节奏 Agent 留出空间）

- [ ] 该章是否有明确的起承转合（叙事骨架完整）？
- [ ] 核心事件是否在本章内解决/推进（不是悬而未决地交给下一章处理）？
- [ ] 章节结构是否清晰（方便节奏 Agent 后续优化断点和钩子）？

> 注：节奏 Agent 负责断章钩子的具体设计，本层只确保编辑修改后的稿件有足够的叙事完整性。

---

## 六、检查输出格式

检查 Agent 检查完成后，输出以下结构（供编辑 Agent 执行修改使用）：

```markdown
## 编辑检查报告

### 该章基本信息
- 章节：第N章
- 字数：XXXX字
- 当前阶段：初稿/修订稿

### 发现的问题

#### 🔴 严重问题（影响阅读）
- [问题描述]
  - 位置：第X段
  - 建议修改：

#### 🟡 中等问题（影响体验）
- [问题描述]
  - 位置：第X段
  - 建议修改：

#### 🟢 轻微问题（优化建议）
- [问题描述]
  - 建议修改：

### 通过的检查项
- ✅ [通过项]

### 总体评价
[是否符合发布标准，给出结论]

### 修改优先级
1. [最优先处理的问题]
2. [次优先]
3. ...
```

## 七、与节奏 Agent 评估标准的分工

| 检查项 | 检查 Agent | 节奏 Agent |
|--------|-----------|------------|
| 设定/人物/结构层/文字层 | ✅ | |
| 叙事骨架完整性（起承转合） | ✅ 定性（有没有） | |
| 爽点（有没有） | ✅ 定性 | |
| 爽点铺垫/释放质量 | | ✅ 定量（好不好）|
| 高潮节奏/情绪曲线 | | ✅ |
| 章节钩子设计/断章位置 | | ✅ |
| 毒点检查 | ✅ | |
| 审核合规（内容层） | ✅ | |
| 叙事完整性（格式层） | ✅ | |
| 标点/错别字/格式 | | ✅ |

> 检查 Agent 只做定性判断（有没有/对不对），节奏 Agent 负责定量优化（好不好/够不够）。
