# 节奏评估标准

> WriteSync（共笔）项目
> 适用阶段：每章校对后作为**人工参考**评估节奏（v0.3.0 后节奏 Agent 已合并入校对 Agent，不再独立调用）
> 当前工作流：文笔 Agent → 校对 Agent（含节奏分析）
> 输入：终稿文本 + 该章章纲

---

## 一、整体节奏评估

### 1.1 章内节奏曲线

检查该章的情绪起伏是否合理：

```
情绪强度 →  峰
            ↑        ★ 爽点/高潮
            ↑     ★        ★  次高潮
            ↑  ★     ★
            ↑ ★   ★
            ↑ ★
            ↑★
情绪强度 → 谷
         ↑              ↑              ↑
         章首            章中           章末（钩子）
```

- [ ] 章节是否有明确的情绪高点（爽点/冲突爆发/关键转折）？
- [ ] 高潮点是否出现在章节中后段（不是开头就是高潮，后继乏力）？
- [ ] 高潮前是否有足够的铺垫（憋屈/压抑/期待）？
- [ ] 章末情绪是否在相对高点结束（留下钩子）或相对低点（制造悬念）？
- [ ] 是否有连续的平缓段落（让读者失去耐心）？

### 1.2 章节节奏类型

根据该章在整体故事中的位置，判断节奏类型：

| 节奏类型 | 特征 | 适用场景 |
|----------|------|----------|
| **爆发型** | 快速推进，事件密集 | 战斗/打脸/突破关键节点 |
| **积蓄型** | 慢节奏，埋设伏笔 | 日常过渡/情感铺垫/修炼 |
| **交替型** | 张弛结合，有快有慢 | 日常中的小冲突 |
| **悬念型** | 持续紧张，不释放 | 危机/追逃/生死关卡 |

- [ ] 该章选择的节奏类型是否符合章纲预设？
- [ ] 该章节奏类型是否与前后章节形成对比（连续两章都是爆发型会疲劳）？

### 1.3 节奏速度

- [ ] 核心事件推进是否够快（不拖沓）？
- [ ] 必要的铺垫是否足够简洁（不冗长）？
- [ ] 场景转换是否干脆（不在场景间反复横跳）？

## 二、爽点设计

> 爽点是网文的核心驱动力，每章至少应有一个明确的爽点。

### 2.1 爽点类型识别

检查该章是否包含以下爽点类型之一：

| 类型 | 描述 | 常见场景 |
|------|------|----------|
| **装逼打脸** | 主角展现实力，打脸轻视自己的人 | 被嘲讽 → 反杀 → 对方震惊 |
| **升级突破** | 境界/实力提升 | 修炼成功、获得传承、突破瓶颈 |
| **收获获取** | 获得物品/功法/人脉/情报 | 捡到宝贝、拜师、发现秘密 |
| **复仇打脸** | 对敌人的反击和报复 | 曾经欺负主角的人得到报应 |
| **逆天改命** | 在绝境中逆转 | 被逼入死角 → 爆发 → 反转 |
| **情感满足** | 爱情/友情/亲情的进展 | 攻略对象示好、兄弟情深 |
| **智谋碾压** | 以智取胜，展现智商 | 布局、反间、算计成功 |

- [ ] 该章是否有明确的爽点？
- [ ] 爽点类型是否符合题材特性（修仙→升级爽点，都市→装逼爽点）？

### 2.2 爽点铺垫

爽点需要铺垫才有力度：

- [ ] 打脸前是否有足够的憋屈/轻视铺垫？
- [ ] 升级前是否有足够的压抑/瓶颈铺垫？
- [ ] 收获前是否有足够的期待/寻找铺垫？
- [ ] 铺垫和爽点之间的情绪反差是否足够大？

### 2.3 爽点释放

爽点写透才有效果：

- [ ] 爽点场景是否充分展开（不是一笔带过）？
- [ ] 对方的反应是否到位（打脸要写出对方的震惊/后悔）？
- [ ] 主角的内心满足是否有体现（不能完全没情绪）？
- [ ] 围观群众的反应是否有体现（衬托效果）？

## 三、章节钩子设计

> 章节钩子是网文日更模式的核心——让读者欲罢不能地追更。

### 3.1 钩子类型

检查章末是否使用了以下钩子类型之一：

| 类型 | 描述 | 示例 |
|------|------|------|
| **悬念钩子** | 留下未解之谜 | "就在他转身的瞬间，袖口滑落一张纸条……" |
| **冲突钩子** | 冲突未解决 | "话音未落，门外传来急促的敲门声" |
| **转折钩子** | 出乎意料的转折 | "他低头看向胸口，那里插着一柄短剑" |
| **期待钩子** | 预告下一个爽点 | "三日后，便是宗门大比之日" |
| **危机钩子** | 制造紧迫感 | "远处，一道黑影正在逼近" |
| **反转钩子** | 揭露关键信息 | "原来这一切，都是他设计的" |

- [ ] 章末是否有钩子？
- [ ] 钩子类型是否适合本章情绪基调（悬念/冲突/期待）？

### 3.2 钩子质量

- [ ] 钩子是否在章节末尾最后 100-300 字内（不能埋太深）？
- [ ] 钩子是否足够具体（不是泛泛的"还有更精彩的故事"）？
- [ ] 钩子是否足够紧迫（给读者"不读下一章不行"的冲动）？
- [ ] 钩子是否与下一章开头衔接自然（不能章末说A，下章从B开始）？

### 3.3 断章位置

断章是艺术，选择断章位置决定了读者的追读欲望：

- [ ] 是否在情绪高点断章（读者兴奋/紧张/期待时）？
- [ ] 是否避免在平缓叙述中断章（读者会失去兴趣）？
- [ ] 是否避免在高潮顶点完全释放后断章（已经爽完了，读者满足感够了）？
- [ ] 最佳断章点：在高潮即将到来的前一刻，让读者自己想象

## 四、高潮节奏

### 4.1 高潮构建

- [ ] 高潮前是否有足够的铺垫（至少一屏文字）？
- [ ] 高潮前的压抑/憋屈是否到位（反差越大爽感越强）？
- [ ] 高潮时是否有足够的篇幅展开（不是三句话写完）？
- [ ] 高潮后是否有合理的收尾（不能戛然而止）？

### 4.2 高潮节奏控制

- [ ] 高潮前的节奏是否逐渐加速（用短句/快节奏推高情绪）？
- [ ] 高潮顶点是否有一句核心描写作为锚点（最燃/最虐/最震撼的那一句）？
- [ ] 高潮后是否有短暂的喘息（情绪不能一直紧绷）？

### 4.3 战斗节奏（如有）

- [ ] 战斗前是否有站位/局势介绍（让读者知道谁强谁弱）？
- [ ] 战斗过程是否有起伏（不能一招秒杀，要有多轮交锋）？
- [ ] 战斗转折是否有铺垫（主角获胜不是靠外挂，是靠前面的伏笔）？
- [ ] 战斗结果是否有围观者反应（衬托主角的强）？

---

## 五、情绪节奏

### 5.1 情绪连贯性

- [ ] 章节内的情绪变化是否有迹可循（不能上一秒愤怒下一秒开心）？
- [ ] 情绪转换是否有合理的触发事件（不是凭空切换）？
- [ ] 角色情绪是否符合其性格（不同性格的人情绪反应不同）？

### 5.2 情绪渲染

- [ ] 关键情绪场景是否有足够的氛围描写（不是干巴巴的陈述）？
- [ ] 情绪是否通过动作/神态/环境多维度展现（不是只用"他很愤怒"）？
- [ ] 网文特有的"系统提示"/"面板显示"是否服务于情绪节奏（不是堆砌数字）？

### 5.3 读者情绪引导

- [ ] 该章是否能调动读者情绪（爽/燃/虐/感动/好奇）？
- [ ] 读者情绪是否被引导向预设方向（不要让读者产生预期外的情绪）？
- [ ] 是否有意识地制造"期待感"（让读者想知道接下来会发生什么）？

## 六、检查输出格式

节奏 Agent 优化完成后，输出以下结构：

```markdown
## 节奏评估报告

### 该章基本信息
- 章节：第N章
- 字数：XXXX字
- 当前阶段：节奏优化
- 该章节奏类型：[爆发型/积蓄型/交替型/悬念型]

### 节奏曲线评估
- 情绪高点位置：章节约XX%处
- 情绪高点类型：[爽点类型]
- 章末情绪：[高点/低点/悬念]
- 整体评价：[节奏是否合理]

### 爽点评估
- 是否有爽点：✅/❌
- 爽点类型：[类型]
- 爽点铺垫：✅/❌
- 爽点释放：✅/❌
- 优化建议：[如有]

### 钩子评估
- 章末是否有钩子：✅/❌
- 钩子类型：[类型]
- 钩子质量评分：⭐⭐⭐⭐⭐
- 断章位置评价：[合理/需调整]
- 优化建议：[如有]

### 高潮评估（如有）
- 高潮铺垫：✅/❌
- 高潮展开：✅/❌
- 高潮节奏：✅/❌
- 优化建议：[如有]

### 修改优先级
1. [最优先处理的问题]
2. [次优先]
3. ...

### 节奏优化后的预期
- 预计字数：XXXX字
- 情绪曲线：[描述优化后的节奏曲线]
- 核心爽点：[优化后该章的核心爽点]
- 章末钩子：[优化后的钩子内容]
```
