# 开篇三章技法

> WriteSync（共笔）项目
> 用途：帮助策划 Agent / 文笔 Agent 设计开篇三章的核心技法参考
> 适用阶段：选题确认后，策划阶段设计开篇；文笔 Agent 写作前参考
> 核心目标：让读者在第三章结束时产生强烈的追读欲望

---

## 一、开篇的核心逻辑

网文读者决定是否继续阅读的决策点：**前三章**。

传统文学可以用慢热开头吸引读者，网文不行——读者是在碎片时间阅读，随时可以关闭。

### 开篇的三个目标

| 目标 | 说明 | 失败后果 |
|------|------|----------|
| **建立代入** | 读者能快速代入主角视角 | 读者当旁观者，无法共情 |
| **展示卖点** | 让读者知道这本书有什么好看的 | 读者不知道值不值得花时间 |
| **制造期待** | 留下钩子，让读者想看下去 | 读者看完第一章就关闭 |

### 开篇的禁忌

- ❌ 用大段环境描写开场（读者不认识这个世界，描写无意义）
- ❌ 用大段设定介绍开场（读者不知道这些设定的意义）
- ❌ 用次要角色的视角开场（读者不知道这个角色是谁）
- ❌ 用与主角无关的事件开场（读者不知道在讲谁）
- ❌ 开篇就虐主且看不到希望（读者不想受这个气）
- ❌ 开篇节奏拖沓（读者在碎片时间，注意力有限）

## 二、第一章：立人设 + 埋钩子

### 2.1 核心任务

第一章的核心任务只有两个：**让读者认识主角 + 留下一个足够强的钩子**。

### 2.2 主角登场三要素

第一章结束时，读者必须知道：

1. **主角是谁**（名字 + 身份 + 核心性格）
2. **主角当前处境**（困境/困境感，让读者产生同情或好奇）
3. **主角有什么特殊之处**（金手指/身份/能力，让读者期待后续）

### 2.3 开篇钩子的常见类型

| 钩子类型 | 描述 | 效果 |
|----------|------|------|
| **冲突型** | 主角面临直接的冲突或威胁 | 最强，读者立刻想知道结果 |
| **悬念型** | 留下未解之谜 | 强，读者想知道真相 |
| **反差型** | 展示主角的某种反差（弱外表下的强内在）| 强，读者想看反转 |
| **金手指型** | 开篇展示主角的特殊能力/机遇 | 中强，读者想知道能力怎么用 |
| **爽点型** | 开篇就来一个小爽点 | 中，读者想看更大的爽点 |
| **情感型** | 展示主角的情感羁绊（爱人/家人/兄弟）| 中，读者想看情感发展 |

### 2.4 第一章结尾的钩子设计

第一章结尾必须有一个钩子，常见写法：

```
- 冲突未解决：主角被欺负/被退婚/被打压，读者想看反击
- 悬念未解开：发现了一个秘密/异常，读者想知道真相
- 机遇刚出现：获得了金手指，读者想知道怎么用
- 危险逼近：敌人出现/危机将至，读者想看如何应对
```

**关键**：第一章结尾的钩子，要在第二章得到回应或升级，不能一直吊着读者。

### 2.5 第一章的常见结构

```
场景1：建立主角当前处境（300-800字）
  → 主角在做什么、日常是什么状态
  → 暗示主角的困境

场景2：冲突爆发或机遇出现（800-1500字）
  → 打破主角的日常
  → 冲突/金手指/秘密 三选一

场景3：主角的第一次应对（500-1000字）
  → 展示主角的性格（面对困境怎么做）
  → 展示金手指/能力（如果有）
  → 留下第二章钩子
```

## 三、第二章：冲突展开 + 加深期待

### 3.1 核心任务

第二章的核心任务：**让冲突升级或深入，展示故事的核心卖点，让读者产生"想知道后续"的强烈欲望**。

### 3.2 常见的第二章写法

| 类型 | 写法 | 适用场景 |
|------|------|----------|
| **冲突升级型** | 第一章的冲突继续发酵，越来越大 | 主角被欺负 → 被更大的势力针对 |
| **反击型** | 主角开始反击，展示能力 | 打脸铺垫 → 第一次打脸 |
| **金手指深入型** | 深入展示金手指的能力和限制 | 获得功法 → 第一次修炼 |
| **世界观展开型** | 通过冲突引出世界观 | 遇到敌人 → 引出宗门/修炼体系 |
| **配角登场型** | 引入重要的配角 | 主角 → 遇到导师/挚友/宿敌 |

### 3.3 第二章的爽点设计

第二章应该有一个**小爽点**，让读者感到"这本书确实好看"。

常见的小爽点：
- 主角用智慧/能力解决了困境
- 获得了一个小收获（物品/信息/盟友）
- 打脸了一个小反派
- 展示了比预期更强的能力

**关键**：第二章的小爽点要足够，但不能太大——留到第三章做更大的爽点。

### 3.4 第二章结尾的钩子设计

第二章结尾的钩子要比第一章更强，可以是：
- 更大的危机
- 更大的机遇
- 揭露更大的秘密
- 关键角色登场

## 四、第三章：第一个高潮 + 建立追读惯性

### 4.1 核心任务

第三章是开篇三章中最重要的一章——**必须有一个明确的高潮**，让读者确信"这本书值得追"。

### 4.2 第一个高潮的设计

第一个高潮应该包含：

1. **足够的铺垫**：让读者知道"要爽了"（但不能太长，读者会不耐烦）
2. **明确的爽点**：主角获得胜利/突破/收获
3. **足够的展开**：爽点要写透，不能一笔带过

### 4.3 铺垫与爽点的比例

| 铺垫部分 | 爽点部分 | 说明 |
|----------|----------|------|
| 30-40% | 60-70% | 铺垫要简洁，爽点要充分 |

**常见错误**：
- ❌ 铺垫太长（读者在等爽点的时候弃书了）
- ❌ 爽点太短（"他一拳打倒对手，赢了。"读者还没爽够）
- ❌ 铺垫和爽点之间没有情绪落差（铺垫的时候不够憋屈，爽的时候不够爽）

### 4.4 高潮的类型

| 类型 | 写法 | 示例 |
|------|------|------|
| **打脸型** | 主角当众打脸轻视自己的人 | 当众击败嘲讽自己的对手 |
| **突破型** | 主角成功突破或获得关键能力 | 成功突破境界/获得功法 |
| **收获型** | 主角获得重要资源或人脉 | 捡到宝贝/拜师成功 |
| **反转型** | 出乎意料的反转 | 原来一切都在主角计划中 |
| **危机型** | 主角在危机中展现出非凡能力 | 被围困时反杀所有人 |

### 4.5 第三章结尾的钩子

第三章结尾的钩子要**建立追读惯性**，让读者产生"明天还想看"的感觉：

- 留下更大的悬念（更大的敌人/秘密/挑战即将出现）
- 建立新的期待（下一个爽点是什么）
- 制造紧迫感（时间限制/生命威胁）

**最佳写法**：在高潮顶点稍前一点断章，让读者自己想象高潮的结果。

### 4.6 第三章的常见结构

```
场景1：铺垫（500-1000字）
  → 制造压抑/憋屈/期待感
  → 敌人出现/对手出现

场景2：高潮爆发（1500-2500字）
  → 主角开始反击/突破/爆发
  → 爽点展开，写透

场景3：收尾 + 钩子（500-1000字）
  → 高潮结束后的短暂喘息
  → 留下下一个钩子
```

## 五、开篇三章的节奏总览

```
第一章：立人设 + 埋钩子
  → 主角登场，建立处境
  → 冲突/机遇出现
  → 留下钩子

第二章：冲突展开 + 加深期待
  → 冲突升级
  → 小爽点
  → 钩子升级

第三章：第一个高潮 + 建立追读惯性
  → 充分铺垫
  → 明确高潮（打脸/突破/收获）
  → 爽点展开 + 下一个钩子

读者反应：
  第1章结尾："有点意思，但还不想追"
  第2章结尾："好像还不错，可以试试"
  第3章结尾："必须追下去！"
```

## 六、常见失败模式与对策

### 失败模式1：开篇信息量过大

**问题**：第一章塞入太多设定/人物/背景，读者记不住。

**对策**：
- 只在读者关心某件事的时候，才介绍相关信息
- 用"主角的视角"来展示世界（主角不知道的，读者也不需要知道）
- 信息要"融化"在情节里，不要单独拎出来讲

### 失败模式2：主角没有危机感

**问题**：主角处境太平淡，没有让读者担心的事情。

**对策**：
- 开篇就要有明确的"如果不解决会有严重后果"的困境
- 让读者知道主角想要什么，但有什么在阻止他
- 可以用"时间限制"增加紧迫感

### 失败模式3：爽点铺垫太长

**问题**：憋屈了5章才爽，读者等不到。

**对策**：
- 第三章必须有明确的小高潮
- 前三章的爽点节奏：第一章钩子 → 第二章小爽 → 第三章大爽
- 即使是长篇铺垫，也要中途给读者一些小甜头

### 失败模式4：节奏拖沓

**问题**：大段内心独白/环境描写/对话，进展缓慢。

**对策**：
- 每500字必须有实质性的情节推进
- 删除不影响情节的描写
- 对话要有信息量，不能废话

## 七、与 Agent 的配合

### 策划 Agent / 文笔 Agent 使用指南
- 写作前参考本文设计开篇（怎么写好）
- 确认选题的核心卖点是否在前三章得到体现
- 在章纲模板中标注前三章的节奏类型和钩子设计
- 避免"开篇禁忌"中的写法
- 章节钩子的详细设计参考《钩子技法》

> 注：本文是**开篇专项技法**，《钩子技法》是**章节钩子通用技法**，《节奏评估标准》是**检查标准**。三者的关系：开篇三章决定怎么开场，章节钩子决定怎么断章，节奏评估决定写得对不对。
