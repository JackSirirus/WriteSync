"""
WriteSync Web UI — 完整交互流程规范

本文档定义了 Web UI 每一步的输入/输出/错误处理。
每个「中断点」都需要前端正确识别并展示对应的按钮选项。

符号说明：
  [LLM] = AI 调用，前端显示"思考中"动画
  [I]   = interrupt，等待用户操作
  [END] = 阶段完成，继续下一个

============================================================
全流程（full）— 24 个步骤
============================================================

=== 策划阶段 ===

Step 0: 用户填写准备表单
  输入: 故事想法 / 平台 / 模型 / 供应商 / 流程模式
  [LLM] POST /api/new → 后台创建 session + 启动 graph
  正确: 返回 session_id + interrupts
  错误: ❌ API 超时 — 前端应显示"网络错误"并允许重试
  错误: ❌ LLM 401/400 — 前端应显示错误信息，允许换模型重试

Step 1: 选题生成
  [LLM] 选题Agent 生成 3-5 个选题建议
  中断文本: "请选择一个选题，或修改需求后重新生成"
  [I] 选项: [确认] / [重新生成] / [退出]
  正确: 用户点"确认" → 继续
  正确: 用户点"重新生成" → 回到 选题Agent
  错误: ❌ 用户不选直接关页面 — session 丢失

Step 2: 用户写一句话
  [I] 中断文本: "请写出您的一句话故事核心（≤15字）"
  或 "您写的一句话是：「xxx」 要修改吗？"
  输入区: 文本输入框
  [I] 选项: [提交] / [确认已有] / [退出]
  正确: 用户输入文字点"提交" → story.step1 被设置
  正确: 用户点"确认已有" → 跳过输入
  错误: ❌ 提交空文本 — 输入为空时需提示
  错误: ❌ 超过 15 字 — 需要截断或警告

Step 3: 一句话确认
  [I] 中断文本: "您的一句话：「xxx」。确认作为故事核心？(y=确认)"
  [I] 选项: [确认] / [再改改] / [退出]
  正确: "确认" → 进入 Step 4
  正确: "再改改" → 回到 Step 2（重新输入）
  正确: "退出" → session 重置

Step 4: 五句话展开
  [LLM] 策划Agent 将一句话展开为五句话
  中断文本: "请确认故事摘要，或提出修改意见"
  [I] 输入区: 修改意见文本框
  [I] 选项: [确认] / [提意见修改] / [退出]
  正确: "确认" → 进入 Step 5
  正确: "提意见修改" + 输入文字 → 重新生成
  正确: "退出"

Step 5: 五段扩展
  [LLM] 扩展Agent 将五句话各自扩成一段
  中断文本: "五句话已展开为五段，请确认或提出修改意见"
  或 "请先生成扩展段落"
  [I] 输入区: 修改意见文本框
  [I] 选项: [确认] / [提意见修改] / [退出]
  正确: "确认" → 进入 Step 6
  正确: "提意见修改" + 文字 → 重新生成

Step 6: 角色设定
  [LLM] 角色Agent 生成角色卡
  中断文本: "请确认角色设定，或提出修改意见"
  或 "请先生成角色设定"
  [I] 输入区: 修改意见文本框
  [I] 选项: [确认] / [提意见修改] / [退出]
  正确: "确认" → 进入 Step 7
  正确: "提意见修改" + 文字 → 重新生成

Step 7: 世界观设定
  [LLM] 世界观Agent
  中断文本: "请确认世界观设定，或提出修改意见"
  或 "请先生成世界观设定"
  [I] 输入区: 修改意见文本框
  [I] 选项: [确认] / [提意见修改] / [退出]
  正确: "确认" → 进入 Step 8
  正确: "提意见修改" + 文字 → 重新生成

Step 8: 叙事概要
  [LLM] 叙事概要Agent
  中断文本: "叙事概要已生成（xxx字）：...... 确认用于章纲生成？(y=确认, n=重新生成)"
  或 "请先生成叙事概要"
  [I] 选项: [确认用于章纲] / [重新生成] / [退出]
  正确: "确认" → 进入 Step 9
  正确: "重新生成" → 重新生成

Step 9: 章纲生成
  [LLM] 章纲Agent
  中断文本: "请确认章纲，或提出修改意见"
  或 "请先生成章纲"
  [I] 输入区: 修改意见文本框
  [I] 选项: [确认] / [提意见修改] / [退出]
  正确: "确认" → 进入 Step 10
  正确: "提意见修改" + 文字 → 重新生成

=== 写作阶段 ===

Step 10: 写作入口
  [LLM] 创建版本快照
  [I] 中断文本: "准备开始写作第 1 章（共 X 章），是否继续？"
  [I] 选项: [开始写作] / [跳过] / [退出]
  正确: "开始写作" → 进入 Step 11
  正确: "跳过" → 流程结束
  错误: ❌ 策划阶段未完成 — "策划阶段未完成，无法进入写作阶段"

Step 11: 写初稿
  [LLM] 文笔Agent 生成一章正文
  [LLM] 文笔检查Agent 检查质量
  [I] 中断文本: "第X章初稿（xxx字）：...... 确认？(y=确认继续, n=重写, m=提意见重写)"
  [I] 输入区: 修改意见文本框
  [I] 选项: [确认继续] / [提意见重写] / [直接重写] / [退出]
  正确: "确认继续" → 进入 Step 12
  正确: "提意见重写" + 文字 → 回到文笔Agent重新生成
  正确: "直接重写" → 回到文笔Agent不传 feedback
  错误: ❌ LLM 超时 — 大篇幅正文可能超时，需 fallback

Step 12: 编辑
  [LLM] 编辑Agent 提出修改建议
  [I] 中断文本: "第X章编辑完成（N条修改建议） 确认？(y=继续节奏优化, n=提意见重新编辑)"

  [I] 选项: [确认继续] / [提意见重新编辑] / [退出]
  正确: "确认继续" → 进入 Step 13
  正确: "提意见重新编辑" + 文字 → 回到编辑Agent

Step 13: 节奏+校对
  [LLM] 节奏Agent + 校对Agent
  [I] 中断文本: "第X章终稿（xxx字）：...... 确认发布？(y=确认, n=从头重写)"
  [I] 选项: [确认发布] / [从头重写] / [退出]
  正确: "确认发布" → 进入 Step 14
  正确: "从头重写" → 回到文笔Agent

Step 14: 多章循环
  [I] 中断文本: "第 X 章完成。是否继续写第 Y 章？"
  [I] 选项: [继续下一章] / [结束] / [退出]
  正确: "继续下一章" → 回到 Step 11（下一章）
  正确: "结束" → 进入 Step 15

Step 15: 全书审查
  [LLM] 全书审查Agent
  [I] 中断文本: "全书审查完成：通过/需修改 结构问题：...... 建议：...... 确认完成？(y=确认, n=标记待修改)"
  [I] 选项: [确认完成] / [标记待修改] / [退出]
  正确: "确认完成" → 流程结束（导出可用）
  正确: "标记待修改" → 结束但保留标记

=== 项目加载流程（v0.2.0+）===

Step L0: 页面初始化
  前端 GET /api/projects 查询已有项目
  显示"设置"面板 vs "已有项目"列表
  后端: PersistenceManager.list_projects() 遍历 projects/ 目录
  正确: 返回项目列表（名称/平台/状态/更新时间）
  正确: 无项目 → 返回空列表 → 前端直接显示设置面板
  错误: ❌ filesystem 错误 — 返回空列表，不影响新项目创建

Step L1: 用户选择项目
  [I] 前端展示项目卡片，每张含：
    - 项目名称
    - 平台（起点/番茄/飞卢/纵横）
    - 当前状态（DRAFTING / COMPLETED）
    - 最后更新时间
    - 简短摘要（如"选题: 程序员修仙 | 已写: 3/20 章"）
  选项: [点击卡片加载] / [新建项目] / [退出]

Step L2: 后端加载项目
  [API] POST /api/load/{project_id}
  后端:
    1. PersistenceManager.load_project() — 从 JSON 重建 WriteSyncState
    2. _build_session() — rebuild graph + config (thread_id = project_id + "_reload")
    3. _run_session() — 后台线程启动 graph.invoke()
    4. 图执行时自动调用 Smart Resume 跳过已完成步骤
       - Agent 节点检查"输出已存在 → 返回 {}"
       - 确认节点检查"confirmed_at → 返回 {}"
       - 路由函数根据状态标记跳转到下一步
    5. 停在第一个未完成的 interrupt 点返回

  正确: 返回 session_id + interrupt + 项目摘要
  错误: ❌ 项目文件损坏 — 返回 404，前端提示"项目加载失败"
  错误: ❌ project_id 不存在 — 返回 404

Step L3: 前端恢复工作台
  前端收到 session_id 后：
    1. 存储 session_id 到内存
    2. 轮询 /api/status/{sid} 获取状态
    3. 根据 interrupt 类型渲染对应的面板
    4. 用户正常操作（确认/修改/继续）

  [LLM] 后续操作与正常流程一致（Step 1-15）

=== 智能跳过机制 ===

Smart Resume 的核心原理：

所有 Agent 节点在执行前检查"我的输出是否已存在"：
  - 如果已存在 → 不调 LLM，直接返回 {}（空更新）
  - 如果不存在 → 正常调 LLM 生成

所有确认节点在执行前检查"我的步骤是否已确认"：
  - 如果已确认 → 不 interrupt，返回 {}
  - 如果未确认 → 正常 interrupt 等用户

路由函数判断依据（completed_steps / confirmed_at / 数据存在性）决定跳转到哪个节点。

效果：加载已完成 80% 的项目，图在 ~50ms 内完成跳过，停在未完成的步骤。

=== 错误场景 ===

E1: LLM 调用超时
  LLM 响应超过 300s，后端线程超时
  前端表现: 长时间显示"思考中"
  处理: 后端 session["error"] 应包含超时信息，前端检测后显示重试按钮

E2: LLM 认证失败 (401)
  前端表现: POST /api/new 返回 error
  处理: 显示错误 + 自动回到设置面板

E3: 网络断开
  前端表现: fetch 抛异常
  处理: try/catch 中显示"网络错误"，保留当前会话

E4: 刷新页面 (session 丢失)
  前端表现: 轮询 /api/status 返回 404
  处理: 检测到 ongoing session 失效 → 显示设置面板 + 提示"会话已过期"

E5: 并发提交
  用户快速双击按钮
  前端表现: 两次 /api/resume 被同时发送
  处理: 按钮点击后立即 disabled，防止重复提交

E6: 空反馈提交
  用户没有输入反馈直接点"提意见修改"
  处理: 允许（空反馈会触发无参重生成）
